## 온라인 증류 (Online Distillation)

Week 3에서 다룬 오프라인 증류(Offline Distillation)는 사전 학습된 교사 모델이 고정된 상태에서 학생 모델을 훈련하는 방식이었다. 온라인 증류는 교사와 학생이 **동시에 학습**하는 방식이다.

### 오프라인 vs 온라인

| 비교 항목 | 오프라인 증류 (Offline) | 온라인 증류 (Online) |
| --- | --- | --- |
| 교사 모델 | 사전 학습 후 고정 | 학습 중 함께 업데이트 |
| 학습 방식 | Teacher → Student 순차 | 동시 학습 |
| 유연성 | Teacher 변경 불가 | Teacher가 계속 개선됨 |

### 지식 증류 방식에 따른 분류 (복습)

Week 3에서 정리한 로짓 기반 증류에 이어, 지식을 어디서 추출하느냐에 따라 세 가지로 나뉜다.

- **로짓 기반 증류(Logit-based)**: 최종 출력(로짓)의 차이로 Distillation Loss 계산
- **피처 기반 증류(Feature-based)**: 로짓 대신 **중간 레이어의 피처 맵**을 비교하여 증류
- **관계 기반 증류(Relational)**: 인스턴스 간의 관계(Instance Relations)를 학습 모델이 모방

### 온라인 증류의 주요 방법

**단일 교사-학생 구조(Single Teacher-student)**

교사 모델과 학생 모델이 동시에 학습하며, 실시간으로 자신의 성능을 개선한다.

**상호 학습 구조(Mutual Learning)**

교사와 학생의 구분이 명확하지 않다. 여러 모델이 서로 교사 역할을 하며 상호 작용한다. 모델들이 서로 예측 값을 참조하며 실시간으로 지식을 공유한다.

### 온라인 증류의 학습 과정

1. **동시 학습**: 교사 모델과 학생 모델이 동시에 학습 진행. 로짓을 기반으로 학생 모델이 교사 모델의 예측 값을 모방한다.
2. **손실 함수 정의**
   - 교사 모델 출력(Distillation Loss): Soft Targets과 학생 모델 출력의 차이 → **KL Divergence** 사용
   - 학생 모델 출력(Student Loss): 학생 모델 출력과 실제 라벨(Hard Labels)의 차이 → **Cross Entropy Loss** 사용
   - 최종 손실 함수: $L = \alpha \cdot L_{\text{Distillation}} + (1 - \alpha) \cdot L_{\text{Student}}$ (α로 두 손실 간 가중치 비율 조절)
3. **지속적인 모델 업데이트**: 학습 도중에 교사와 학생 모델이 지속적으로 업데이트된다. 교사 모델이 더 나은 성능을 발휘하면 학생 모델도 더 나은 성능을 발휘한다.

### 장점

1. 실시간 지식 전달
2. 동시 학습에 따른 상호 보완 효과
3. 복잡한 모델 학습 필요 없음
4. 자유로운 모델 구조 설계

### 단점

1. 복잡한 학습 과정 (교사와 학생이 동시에 학습)
2. 안정성 문제 (잘못된 지식 학습 가능성)

### 적용 예시

- **컴퓨터 비전**: 큰 교사 모델이 학습하는 동안 작은 학생 모델이 동시에 학습
- **자연어 처리(NLP)**: BERT와 같은 대형 언어 모델 → 작은 언어 모델로 실시간 학습. 경량화된 모델로도 성능 유지

## 셀프 증류 (Self Distillation)

하나의 모델 내에서 스스로 지식을 증류하는 방식이다. 외부의 교사 모델 없이, 단일 모델이 교사 모델(Teacher)과 학생 모델(Student)을 동시에 수행한다.

별도의 모델이 필요하지 않고, 중간 계층 또는 이전 예측을 사용하여 더 정제된 예측을 학습한다.

### 주요 개념

1. **단일 모델의 교사-학생 관계**: 같은 모델 내에서 더 깊은 레이어의 예측을 기준으로 얕은 레이어의 출력을 학습하게 한다.
2. **중간 레이어 예측 활용**: 중간 레이어 또는 이전 레이어의 예측 값과 최종 출력을 비교하여, 중간 레이어가 더 정교한 예측을 하도록 학습한다.
3. **지식 정제**: 중간 예측을 후반부 레이어에서 더 정제된 형태로 만들어 다시 학습한다. 학습의 깊이를 더하고 더 높은 일반화 성능을 얻을 수 있다.

### 학습 과정

1. **모델의 다중 예측 생성**: 동일한 입력 데이터를 처리하여 여러 계층에서 나온 예측 값을 생성. 중간 레이어 또는 특정 레이어에서 예측 값이 정교하지 않은 상태이다.
2. **교사-학생 관계 설정**: 최종 레이어 출력을 교사 예측으로, 초기/중간 레이어 출력을 학생 예측으로 사용한다. 두 출력을 비교하여 중간 예측 값이 더 나은 예측을 하도록 학습한다.
3. **손실 함수 정의**
   - Distillation Loss: 중간 레이어(학생)와 최종 레이어(교사)의 출력 차이 → **KL Divergence** 또는 **L2 손실** 사용
   - Student Loss: 학생 모델 출력과 실제 라벨(Hard Labels) → **Cross Entropy Loss** 사용
   - 최종 손실 함수: $L = \alpha \cdot L_{\text{Distillation}} + (1 - \alpha) \cdot L_{\text{Student}}$ (α로 두 손실 간의 균형 조정)

### 장점

1. 모델 성능 개선
2. 추가적인 교사 모델 없이 효율성 개선
3. 모델 압축 없이 성능 향상
4. 깊은 학습 계층의 정보 활용

### 단점

1. 성능 향상의 한계
2. 계층 간 상호 작용의 복잡성

### 적용 예시

- **컴퓨터 비전**: 이미지 분류 작업에서 셀프 증류를 사용. 중간 레이어 예측 값을 최종 예측 값과 비교하며 성능을 개선한다.
- **자연어 처리(NLP)**: BERT와 같은 대형 언어 모델에서 중간 계층의 임베딩을 최종 출력과 비교하여 학습. 모델 성능 개선에 활용한다.

| 방법 | 특징 | 적합한 상황 |
| --- | --- | --- |
| 오프라인 증류 (Offline) | 학습 단계 분리, 여러 학생 모델 실험 가능 | 모델 성능 유지하며 경량화된 모델 배포 시 |

## NAS 개요 (Neural Architecture Search)

### NAS란?

신경망 아키텍처를 **자동으로 설계**하는 기술이다.

### NAS의 필요성

신경망 설계의 복잡성이 증가하고 있다. 레이어 수, 활성화 함수, 연결 방법 등 다양한 설계 요소가 있으며, 대형 데이터셋이나 복잡한 문제에서 최적의 아키텍처를 찾는 것은 많은 노력이 필요하다.

- **경쟁력 있는 성능 향상**: 수동으로 설계된 신경망을 뛰어넘는 성능 제공
- **자원 제약 환경에서 최적화된 네트워크**: 모바일 장치, 임베디드 시스템과 같은 자원 제약 환경에서 효율적으로 작동하는 경량화된 신경망 설계

## NAS 구성요소

NAS는 세 가지 핵심 구성요소로 이루어진다.

1. **탐색 공간(Search Space)**: NAS가 탐색할 수 있는 가능한 신경망 구조들의 집합
2. **탐색 전략(Search Strategy)**: 탐색 공간 내에서 최적의 아키텍처를 찾기 위해 탐색을 수행하는 방법
3. **성능 추정 전략(Performance Estimation Strategy)**: 선택된 아키텍처의 성능을 평가하는 방법

이 세 가지가 **순환적으로 상호작용**한다: 탐색 전략이 탐색 공간에서 아키텍처(A)를 선택 → 성능 추정 전략이 A를 평가 → 평가 결과를 탐색 전략에 피드백 → 더 나은 아키텍처를 탐색.

### 탐색 공간 (Search Space)

NAS가 탐색할 수 있는 가능한 신경망 구조들의 집합이다.

주요 요소:

- 신경망층의 유형: 컨볼루셔널 레이어, 풀리 커넥티드 레이어, 풀링 레이어
- 활성화 함수: ReLU, Sigmoid, Tanh 등
- 신경망의 깊이 (레이어 개수)
- 각 층의 채널 수 및 필터 크기
- 네트워크 연결 방식: 스킵 연결, 병렬 연결
- 크기: 탐색 공간이 너무 크면 최적의 아키텍처를 찾는 데 시간이 오래 걸림. 너무 작으면 좋은 성능의 모델을 찾기 어려움.

> **셀 기반 설계(Cell-based Design)**: 셀(Cell)이라는 작은 신경망 모듈을 탐색하고, 반복해 큰 네트워크를 구성한다. 전체 네트워크 복잡성을 줄여서 탐색 공간을 보다 더 효율적으로 축소할 수 있다.

탐색 공간의 유형:

- **연속적 탐색 공간**: 미분 가능한 나스(DARTS) — 그래디언트 기반 최적화로 빠르게 탐색
- **이산적 탐색 공간**: 진화 알고리즘 기반 나스 — 교배/돌연변이를 통해 세대별 탐색

### 탐색 전략 (Search Strategy)

탐색 공간 내에서 최적의 아키텍처를 찾기 위해 탐색을 수행하는 방법이다.

**탐색 전략 유형:**

- **무작위 탐색(Random Search)**: 계산 비용이 적고 베이스라인으로 사용. 효율적으로 찾는 데 한계가 있다.
- **진화 알고리즘(Evolutionary Algorithms)**: 유전 알고리즘과 비슷하게, 초기 아키텍처 집합을 설정한 후 좋은 성능 아키텍처를 선택. 교배와 돌연변이를 통해 새로운 아키텍처를 생성하며 점차 최적화. 예) AmoebaNet.
- **강화 학습(Reinforcement Learning, RL)**: 에이전트가 아키텍처를 생성하고 성능 피드백에 따라 선택을 학습. NAS는 높은 성능을 도출할 수 있지만 계산 비용이 매우 클 수 있다. 예) NASNet.
- **그리디 탐색(Gradient-based Search)**: 미분 가능한 아키텍처 매개변수 공간을 정의하고, 그래디언트 기반 최적화 방법으로 탐색. 예) DARTS.

**탐색 전략의 선택 기준:**

| 기준 | 내용 |
| --- | --- |
| 탐색 효율성 | 적은 자원으로 빠르게 최적의 아키텍처 찾기 |
| 계산 비용 | 강화학습/진화 알고리즘: 자원 소모 큼. 그리디 탐색: 상대적으로 효율적 |
| 성능 | 최종 선택 아키텍처의 성능. 데이터셋과 문제의 전이가 가능한지에 따라 선택 |

### 성능 추정 전략 (Performance Estimation Strategy)

선택된 아키텍처의 성능을 평가하는 방법이다. 설정 방식에 따라 NAS의 효율성이 달라진다.

| 방법 | 내용 |
| --- | --- |
| 완전 훈련 (Full Training) | 선택된 아키텍처를 끝까지 훈련 후 성능 평가. 정확하지만 시간/자원 소모 큼 |
| 조기 종료 (Early Stopping) | 훈련 도중 성능이 일정 기준 도달 시 중단. 시간 절약되지만 정확성 한계 |
| 가중치 공유 (Weight Sharing) | 여러 아키텍처 훈련 시 동일한 가중치 공유, 빠른 선택. ENAS가 대표 사례. 자원 절약되지만 정확한 성능 추정 불가 |
| 프록시 모델 (Proxy Model) | 전체 데이터셋 대신 작은 데이터셋, 간단 문제로 성능 추정. 빠르지만 실제와 다를 수 있음 |

**선택 기준**: 정확성 / 효율성 / 자원 제약 (모바일, 임베디드 환경에서 효율적인 성능 추정 방법 필요)

## NAS 알고리즘 예시

### NASNet

강화 학습 기반의 NAS 방식이다. 강화 학습 에이전트가 작은 셀(Cell)을 설계하고, 이를 쌓아 전체 네트워크를 구성한다.

- 컨트롤러가 샘플 아키텍처를 생성 → 아키텍처를 트레이닝 → Accuracy를 받아와서 컨트롤러에 전달 → 더 좋은 아키텍처를 탐색하는 순환 구조
- ImageNet에서 매우 높은 정확도 달성
- 장점: 성능이 매우 우수, 다양한 데이터셋에 쉽게 전이 가능
- 단점: 많은 계산 자원 소모, 처음 탐색에 수천 개의 GPU 시간 필요

### AmoebaNet

셀 기반 구조 탐색에 **진화 알고리즘**을 사용한다. 교배(Crossover)와 돌연변이(Mutation)로 아키텍처를 진화시킨다.

- ImageNet에서 높은 성능 달성
- 장점: 단순하고 직관적인 방식으로 동작
- 단점: 계산 자원이 많이 소모됨. 탐색 과정이 강화 학습보다 긴 시간이 필요할 수 있음

### ENAS (Efficient NAS)

NASNet의 한계를 극복하고자 만들어졌다. **가중치 공유** 기법으로 탐색 비용을 대폭 절감한다.

- 장점: 적은 계산 자원으로 비슷한 성능을 낼 수 있음
- 단점: 성능 추정의 정확도가 낮을 수 있음. 아키텍처 간의 상호 의존성이 생길 수 있음

### DARTS (Differentiable Architecture Search)

**미분 가능한 탐색** 방식을 도입한다. 탐색 공간을 연속적으로 정의하고 그래디언트 기반 최적화로 빠르게 아키텍처를 탐색한다.

- 장점: 빠르게 최적의 아키텍처를 찾음
- 단점: 국지적 최적값에 빠질 위험 존재. 탐색 공간이 큰 경우 한계 존재

### ProxylessNAS

NAS에서 일반적으로 사용되는 **프록시 모델을 생략**한다. 실제 데이터와 하드웨어에서 직접 아키텍처를 탐색한다.

- 자원 제한적인 하드웨어에서 실행할 수 있는 아키텍처 찾기에 유용
- 장점: 프록시 모델 없이 가능한 아키텍처 탐색
- 단점: 복잡한 탐색 과정 및 많은 시간 소요

### MnasNet

목표: **모바일 장치에서의 효율적인 성능 최적화**. 다중 목표 최적화(Multi-objective Optimization)를 사용하여 속도 및 정확도를 동시에 최대화한다. 강화 학습 사용. ProxylessNAS와 유사한 접근.

- 장점: 실제 모바일 환경에서 높은 효율성과 성능을 보장
- 단점: 계산 비용이 높음. 다양한 하드웨어 환경에서의 최적화를 고려해야 함

### NAS 알고리즘 비교

| 알고리즘 | 탐색 전략 | 핵심 특징 | 적합한 상황 |
| --- | --- | --- | --- |
| NASNet | 강화 학습 | 셀 기반, 높은 정확도 | 성능 우선, 자원 여유 |
| AmoebaNet | 진화 알고리즘 | 교배/돌연변이 기반 | 직관적 탐색 |
| ENAS | 강화 학습 + 가중치 공유 | 비용 대폭 절감 | 제한된 자원에서 빠른 탐색 |
| DARTS | 그래디언트 기반 | 미분 가능, 연속적 탐색 | 빠른 탐색 필요 시 |
| ProxylessNAS | 직접 탐색 | 프록시 없이 하드웨어에서 직접 | 모바일/임베디드 환경 |
| MnasNet | 강화 학습 + 다중 목표 | 속도+정확도 동시 최적화 | 모바일 배포 |

## NAS의 한계와 최신 연구 동향

### NAS의 한계

1. **높은 계산 비용**: 탐색 과정에서 수많은 아키텍처를 생성하고 평가. 가중치 공유, 미분 가능한 NAS 등으로 완화 시도.
2. **탐색 공간의 크기와 복잡성**: 셀 기반 구조(Cell-based Architecture)와 하이퍼파라미터 최적화 기법으로 대응.
3. **탐색 공간의 크기와 복잡성 (성능 추정)**: 조기 종료, 가중치 공유 → 잘못된 아키텍처 선택 가능성 존재.
4. **국지 최적화에 빠질 가능성**: 현재 탐색된 아키텍처에 머무를 수 있음. 다양한 탐색 전략, 초기 탐색 공간의 설정 다양화로 대응.
5. **현실적인 제약 조건 반영 어려움**: ProxylessNAS, MnasNet 등이 이를 해결하려는 시도.

### 최신 연구 동향

- **효율적인 NAS(Efficient NAS)**: ENAS(가중치 공유로 비용 절감), ProxylessNAS(직접 탐색, 모바일 환경)
- **미분 가능한 NAS(Differentiable NAS)**: DARTS — 탐색 공간을 미분 가능하게 만들어 그래디언트 기반 최적화로 빠르게 탐색
- **하이브리드 탐색 전략**: 강화 학습, 진화 알고리즘, 그리디 탐색을 결합
- **자원 제약 환경에서의 NAS**: MnasNet(다중 목표 최적화, 성능과 연산 비용 동시 고려), Hardware-Aware NAS(특정 하드웨어의 제약 조건 반영)
- **NAS와 AutoML의 결합**: 데이터 전처리, 모델 선택, 하이퍼파라미터 최적화까지 자동화된 머신러닝 파이프라인

## NAS 구현 코드

### AutoKeras

딥러닝 모델을 자동으로 설계하고 최적화하기 위한 AutoML 도구이다. 고성능 딥러닝 모델을 쉽게 구축할 수 있는 환경을 제공하며, 복잡한 신경망 설계 과정을 단순화한다.

Image Classification, Image Regression, Text Classification, Text Regression 등을 지원한다.

### DARTS 구현

DARTS 알고리즘의 코드를 구현하여 아키텍처 서치 과정과 평가(Architecture Evaluation)를 직접 실행해 볼 수 있다.
