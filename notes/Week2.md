## Pruning 실전 고려 사항

Pruning의 방법론(Granularity, Criterion, When, How much)을 정했다면, 실제로 적용할 때 추가로 고려해야 할 점들이 있다. 얼마나 많이 제거해도 성능이 유지되는지, 언제 Finetuning을 넣을지, 어떤 프레임워크를 사용할지가 여기에 해당한다.

### Pruning의 실측 효과

실제 모델에 Pruning을 적용했을 때 파라미터 수가 얼마나 줄었는지는 다음과 같다.

| 모델 | Before Pruning | After Pruning | 감소 배율 |
|---|---|---|---|
| AlexNet | 61M | 6.7M | 9X |
| VGG-16 | 138M | 10.3M | 12X |
| GoogleNet | 7M | 2.0M | 3.5X |
| ResNet50 | 26M | 7.47M | 3.4X |

최소 3배에서 최대 9~12배까지 파라미터 수를 줄일 수 있다. 모델 구조에 따라 Pruning 효율이 크게 달라진다는 점을 알 수 있다.


### Pruning Ratio와 Accuracy

Pruning 비율(Pruning Ratio)을 높일수록 파라미터는 줄지만, 어느 시점을 넘으면 정확도(Accuracy)가 급격히 떨어진다. 또한 레이어마다 Pruning에 대한 민감도가 다르므로, 레이어별로 정확도가 떨어지는 시점과 그 영향을 고려해서 Pruning 비율을 결정해야 한다.


### Finetuning의 중요성

Pruning 후 Finetuning을 수행하면 그렇지 않은 경우보다 정확도 손실이 훨씬 작다. 또한 Pruning 횟수(반복 여부)는 Pruning 비율에 따른 정확도 저하를 고려해서 결정해야 한다. Iterative Pruning 방식에서 중간중간 Finetuning을 삽입하는 이유가 바로 이것이다.

### TensorFlow

구글 브레인(Google Brain)팀이 개발한 오픈소스 머신러닝 프레임워크다.

- **기본 데이터 구조**: 다차원 배열(Tensor)을 사용하며, 계산 그래프 형태로 연산이 표현된다.
- **Keras API**와 함께 사용해 간단한 코드로 복잡한 모델 구축이 가능하다.
- **TensorFlow Lite**, **TensorFlow.js**를 통해 모바일 디바이스와 웹 브라우저도 지원한다.
- 분산 컴퓨팅을 통한 대규모 데이터 처리가 가능하며, 연구·상업적 목적 모두에 쓰인다.

### PyTorch

- **동적 계산 그래프**: 코드가 실행될 때마다 그래프가 동적으로 생성된다. 조건문이나 반복문이 자연스럽게 구현되고 디버깅이 용이하다.
- **Python 친화적 인터페이스**: NumPy와 높은 호환성을 가지며, Python 데이터 구조와 쉽게 통합된다.
- 커뮤니티와 튜토리얼이 풍부해 학습과 개발이 비교적 용이하다.
- 컴퓨터 비전, 자연어 처리, 강화 학습 등 실제 제품 환경에서 널리 사용된다.

## 모델 연산량 측정: FLOPs와 MACs

Pruning 적용 전후의 모델 경량화 효과를 수치로 확인하려면 연산량을 측정해야 한다.

대표적인 지표가 **FLOPs**와 **MACs**다.

### FLOPs (Floating Point Operations)

모델에서 발생하는 **곱셈과 덧셈 연산의 총 횟수**다. 네트워크 레이어에서 발생하는 모든 부동소수점 연산을 합산해 계산한다.

레이어 종류별 FLOPs 계산 방법은 다음과 같다.

**완전 연결 계층(Fully Connected Layer)**

$$\text{FLOPs} = 2 \times \text{Input Size} \times \text{Output Size}$$

FC 레이어는 이전 레이어의 모든 뉴런이 다음 레이어의 모든 뉴런과 연결된다.

출력 뉴런 하나를 만들려면, 입력 뉴런 각각에 가중치를 곱하고(Input Size번) 그 결과를 모두 더해야 한다(Input Size번).

즉, 출력 뉴런 하나당 **2 × Input Size** FLOPs가 필요하다.

이 연산을 Output Size개의 출력 뉴런 전체에 반복하면, 전체 FLOPs = **2 × Input Size × Output Size**가 된다.

**합성곱 계층(Convolutional Layer)**

$$\text{FLOPs} = 2 \times F_H \times F_W \times C_{in} \times C_{out} \times H_{out} \times W_{out}$$

- $F_{H}:$ Filter Height, $F_{W}:$ Filter Width
- $C_{in}:$ In Channels, $C_{out}:$ Out Channels
- $H_{out}:$ Output Height, $W_{out}:$ Output Width

하나의 필터는 크기 $F_H \times F_W$이고, 입력 채널 전체($C_{in}$)에 걸쳐 적용된다.

출력 Feature Map의 한 위치(픽셀 하나)를 계산하려면, 필터와 입력 영역 사이의 원소별 곱셈+덧셈이 $F_H \times F_W \times C_{in}$번 발생한다.

FC와 마찬가지로 곱셈+덧셈 한 쌍이 한 번의 FLOPs이므로, 위치 하나당 **2 × $F_H \times F_W \times C_{in}$** FLOPs가 든다.

이 연산이 출력 Feature Map의 모든 위치($H_{out} \times W_{out}$)에서 반복되고, 필터가 $C_{out}$개이므로 전체 수식이 나온다.

**순환 계층(Recurrent Layer, RNN)**

$$\text{FLOPs} = 2 \times \{(\text{Input Size} \times \text{Hidden Size}) + (\text{Hidden Size} \times \text{Hidden Size})\}$$

RNN은 현재 시각의 입력($x_t$)과 이전 시각의 은닉 상태($h_{t-1}$), 두 가지를 입력으로 받는다.

따라서 가중치 행렬이 두 개 존재한다. 하나는 입력→은닉(크기 Input × Hidden), 다른 하나는 은닉→은닉(크기 Hidden × Hidden)이다.

각 행렬 곱셈은 FC 레이어와 동일하게 곱셈+덧셈 쌍이 발생하므로, 두 행렬의 FLOPs를 합산하면 위 수식이 된다.

LSTM의 경우 게이트(input, forget, output, cell gate) 4개가 추가되므로 더 복잡해진다.

**FLOPs 계산 예제**

FC 레이어에서 다음 조건을 가정한다.

- Input Size: 512
- Output Size: 256

$$\text{FLOPs} = 2 \times 512 \times 256 = 262{,}144$$

즉, 이 레이어 하나를 통과할 때마다 약 26만 번의 부동소수점 연산이 발생한다. 참고로 MACs로 환산하면 $262{,}144 \div 2 = 131{,}072$ MACs다.

### MACs (Multiply-Accumulate Operations)

**곱셈과 덧셈을 한 쌍으로 묶어서 계산하는 지표**다. 딥러닝 모델에서는 FLOPs보다 MACs가 더 많이 사용된다.

FLOPs는 곱셈과 덧셈을 각각 1번씩, 즉 2번으로 세고, MACs는 이 쌍을 1번으로 센다. 그래서 MACs는 FLOPs의 절반이다.

$$1 \text{ MACs} = 2 \text{ FLOPs}$$

레이어별 MACs 공식도 FLOPs 공식에서 앞의 2를 제거하면 그대로 나온다.

**완전 연결 계층**

$$\text{MACs} = \text{Input Size} \times \text{Output Size}$$

**합성곱 계층**

$$\text{MACs} = F_H \times F_W \times C_{in} \times C_{out} \times H_{out} \times W_{out}$$

**순환 계층(RNN)**

$$\text{MACs} = (\text{Input Size} \times \text{Hidden Size}) + (\text{Hidden Size} \times \text{Hidden Size})$$

Pooling이나 활성화 함수(Activation Function)는 연산 복잡도에 포함되지 않으므로 이 계산에서 제외한다.

**MACs 계산 예제**

합성곱 계층에서 다음 조건을 가정한다.

- 입력 크기: 32 × 32, 채널 수: 3개
- 필터: 64개의 3 × 3 커널
- 출력 크기: 30 × 30

$$\text{MACs} = 3 \times 3 \times 3 \times 64 \times 30 \times 30 = 1{,}555{,}200$$

FLOPs로 환산하면 $1{,}555{,}200 \times 2 = 3{,}110{,}400$ FLOPs다.

## Quantization 개요

Pruning이 연결(connection) 자체를 제거하는 방식이라면, **Quantization(양자화)** 은 값의 표현 정밀도를 낮추는 방식이다. 파라미터 수는 그대로이지만, 각 값을 저장하는 비트 수를 줄인다.

### Quantization이란?

신경망 모델에서 사용하는 **부동소수점(Floating Point) 연산을 더 낮은 비트의 정수 연산으로 변환**하는 최적화 기법이다. 모델의 정확도를 최대한 유지하면서 연산량과 메모리 사용량을 줄이는 것이 목표다.

예를 들어 32비트 부동소수점(FP32)을 8비트 정수(INT8)로 변환하면, 값의 범위 $[-MAX, MAX]$가 $[-127, 127]$로 매핑된다.


### Quantization의 필요성

Quantization이 필요한 이유는 네 가지다.

- **모델 경량화**: 하드웨어 자원이 제한된 환경에서 메모리 사용량을 줄이고 계산 효율을 높인다.
- **추론 속도 향상**: 시간이 오래 걸리는 부동소수점 연산을 정수 연산으로 바꿔 추론 속도를 크게 높일 수 있다.
- **메모리 사용 감소**: 정수형 데이터는 부동소수점보다 적은 메모리 공간을 사용한다. 대역폭(Bandwidth) 절약도 가능하다.
- **전력 소비 효율화**: 위 효과들로 인해 전력 소비가 함께 줄어든다.

### Quantization의 종류

Quantization은 양자화를 언제 적용하느냐에 따라 크게 세 가지로 나뉜다.

- **Post-training Quantization (PTQ, 사후 양자화)**: 훈련이 끝난 후 모델에 양자화를 적용하는 방법이다. 추가 훈련이 필요 없어 간편하다. Dynamic Range Q, Full Integer Q, Float16 Q로 세분된다.
- **Quantization-Aware Training (QAT, 훈련중 양자화)**: 훈련 과정에서 양자화의 영향을 시뮬레이션하며 학습하는 방법이다. PTQ보다 정확도 손실이 적다.
- **Dynamic Quantization (동적 양자화)**: 추론 중에 실시간으로 가중치를 양자화하는 방법이다. 주로 순환 신경망(RNN, LSTM, GRU)에 사용된다.

## Post-training Quantization (PTQ)

PTQ는 이미 훈련이 완료된 모델에 추가 학습 없이 양자화를 적용하는 방법이다. 적용 시점이 훈련 이후이므로 간편하게 쓸 수 있다는 것이 최대 장점이다. PTQ는 양자화 대상과 정밀도에 따라 세 가지 방식으로 나뉜다.

### Dynamic Range Quantization

**가중치(weight)만 INT8로 양자화하고, 활성화(activation) 값은 FP32로 유지**하는 방식이다. 성능 최적화와 양자화로 인한 성능 저하 최소화를 동시에 노린다.

- **장점**: 기존 모델에 쉽게 적용 가능. 메모리 절약. 활성화가 FP32로 유지되므로 성능 저하가 미미하다.
- **단점**: 활성화도 양자화하는 Full Integer 방식보다 속도 향상이 크지 않다. 극단적인 경량화가 필요한 경우엔 부적합하다.
- **적용 과정**: ① 모델 훈련 완료 → ② 가중치 INT8 변환(활성화는 FP32 유지) → ③ 성능 테스트 → ④ 배포
- **사용 사례**: 모바일 장치, CPU 기반 추론, 임베디드 시스템

### Full Integer Quantization

**가중치와 활성화 값을 모두 INT8로 변환**하는 방식이다. 모든 연산이 정수 연산으로 처리되어 메모리 사용량과 연산 속도를 크게 최적화할 수 있다.

- **장점**: 메모리 사용량 약 4배 감소(FP32 → INT8). 모든 연산이 정수이므로 추론 속도가 크게 향상된다. 전력 소비도 줄어 모바일·엣지 디바이스에 유리하다.
- **단점**: 모든 값이 정수로 변환되기 때문에 정밀도 손실 위험이 높다. 양자화에 민감한 모델이나 세밀한 계산이 필요한 작업에서 성능 저하가 발생할 수 있다. 정밀한 튜닝이 필요하다.
- **적용 과정**: ① 모델 훈련 완료(FP32) → ② 가중치+활성화 모두 INT8 변환(최대·최솟값 기준 스케일링) → ③ 테스트 및 미세 조정 → ④ 배포
- **사용 사례**: 모바일 및 임베디드 기기, 엣지 컴퓨팅, 클라우드 기반 서비스

### Float16 Quantization

**FP32를 FP16(16비트 부동소수점)으로 변환**하는 방식이다. 정수가 아닌 부동소수점 형태를 유지하기 때문에, INT8 기반 양자화보다 정밀도 손실이 적다.

- **장점**: 메모리 사용량이 절반으로 줄어든다. FP16의 빠른 연산으로 추론 속도가 개선된다. GPU(NVIDIA Tensor Core 등)와 TPU에서 FP16 연산을 하드웨어 차원에서 지원해 성능 향상 효과가 크다. 정밀도 손실이 적어 고정밀 연산이 필요한 작업에도 적합하다.
- **단점**: 일부 정밀도 손실이 발생할 수 있다. GPU·TPU 특수 하드웨어 최적화에 의존하므로 CPU 환경에서의 속도 향상은 크지 않다. INT8보다 메모리 절감 효과가 적다.
- **적용 과정**: ① 모델 훈련 완료(FP32) → ② FP16 양자화 적용 → ③ 모델 테스트 및 튜닝 → ④ 배포
- **사용 사례**: 대형 모델 경량화, GPU 기반 실시간 추론, 하드웨어 가속기와의 호환이 중요한 환경

## Quantization-Aware Training (QAT)

PTQ는 훈련이 끝난 모델에 사후적으로 양자화를 적용하기 때문에, 정밀도 손실이 발생할 수 있다. **QAT는 훈련 과정 자체에 양자화의 영향을 반영**하여 이 문제를 완화한다.

### QAT란?

딥러닝 모델을 훈련할 때, **INT8 정수 연산으로 양자화되었을 때의 상황을 시뮬레이션**하며 학습하는 기법이다. 이렇게 하면 모델이 낮은 비트의 정밀도에서도 잘 동작하도록 적응한다.

- **양자화 시뮬레이션**: 훈련 중 정수형 연산을 시뮬레이션하여 양자화 상태를 반영한다.
- **정수 연산을 고려한 훈련**: 가중치, 활성화 함수 출력 등을 양자화 상태로 변환하고, 정수 연산의 한계에 맞게 학습한다.
- **정확도 유지**: 양자화를 훈련 과정에서 반영하기 때문에 PTQ보다 성능 손실이 적다. *QAT > PTQ*
- **장점**: 정밀도 손실 최소화. 실제 하드웨어 환경(정수 연산)을 반영한 훈련으로 배포 환경에서 예상한 성능 발휘. 모델 경량화 효과도 있다.
- **단점**: 양자화 시뮬레이션이 추가되어 훈련 시간이 더 오래 걸린다. 훈련 과정이 복잡해진다. 정수 연산을 지원하는 하드웨어에 최적화된 모델에만 적합하다.
- **적용 과정**: ① FP32로 기본 모델 훈련 → ② 양자화 연산 시뮬레이션(가중치·활성화를 양자화 상태로 변환하여 학습) → ③ 모델 미세 조정(양자화로 인한 성능 저하 최소화) → ④ 양자화 모델 배포
- **주요 활용 사례**: 모바일 및 임베디드 장치, 실시간 추론, 클라우드·엣지 컴퓨팅

### QAT vs PTQ 비교

| 비교 항목 | QAT (훈련중 양자화) | PTQ (사후 양자화) |
|---|---|---|
| **적용 시점** | 모델 훈련중 양자화를 반영 | 훈련 후 양자화를 적용 |
| **정확도** | 정확도 손실이 적고 성능 유지가 잘 됨 | 정확도 손실이 발생할 수 있음 |
| **훈련 시간** | 더 많은 훈련 시간이 필요 | 추가 훈련 없이 바로 적용 가능 |
| **복잡성** | 양자화된 연산을 반영한 훈련 과정으로 더 복잡 | 간단한 양자화 과정으로 구현이 쉬움 |
| **성능 최적화** | 정수 연산 하드웨어에 최적화된 성능 제공 | 정확도 저하가 발생할 가능성이 있음 |

## Dynamic Quantization

**Dynamic Quantization(동적 양자화)** 은 모델 가중치를 추론 시점에 실시간으로 INT8로 양자화하는 방법이다. 가중치는 FP32로 저장되어 있다가, 추론 중 필요할 때만 동적으로 INT8로 변환된다.

### Dynamic Quantization이란?

- **실시간 양자화**: 추론 중 필요할 때만 가중치를 INT8로 양자화한다. 가중치는 FP32로 저장되어 원본 값이 유지되고, 추론 중에만 변환된다.
- **활성화 값은 FP32 유지**: 중간 레이어의 출력은 FP32로 유지되어 정밀도 손실을 최소화한다.
- **적용 대상**: 주로 순환 신경망(RNN, LSTM, GRU)에서 많이 사용된다. 연속적인 입력 데이터를 처리하는 모델에 적합하기 때문이다.

- **장점**: 추론 시 가중치가 INT8이므로 메모리 사용량이 줄고 연산 속도가 빨라진다. PTQ보다 적용이 간단하다. 활성화 값이 FP32로 처리되어 성능 저하가 최소화된다.
- **단점**: 부동소수점 연산이 일부 남아있어 최대 성능 최적화는 불가하다. 가중치를 INT8로 변환할 때 일부 모델에서 작은 성능 손실이 발생할 수 있다. 성능 최적화가 주로 하드웨어 가속기 환경에서 효과적이다.
- **적용 과정**: ① 모델 훈련 완료(FP32) → ② 동적 양자화 적용 설정(추론 중 가중치 FP32 → INT8 변환) → ③ 모델 테스트 및 튜닝 → ④ 배포
- **사용 사례**: RNN/LSTM 모델, 모바일 기기나 IoT 장치에서의 실시간 추론, 클라우드 서비스에서의 메모리 절약
